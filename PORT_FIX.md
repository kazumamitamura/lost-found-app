# ポート3000の競合を解決する方法

## 現在の状況
- ポート3000がプロセスID 34020で使用中（前のアプリの可能性）
- 新しいサーバーはポート3001で正常に起動している
- ブラウザは`http://localhost:3000`にアクセスしようとしている

## 解決方法

### 方法1: ポート3001にアクセス（最も簡単）

ブラウザのアドレスバーを以下のように変更：
```
http://localhost:3001
```

これで即座にアプリが表示されます。

### 方法2: ポート3000を解放する

#### ステップ1: プロセス34020を停止

PowerShellまたはコマンドプロンプトで以下を実行：

```powershell
# プロセスを停止
Stop-Process -Id 34020 -Force
```

または、タスクマネージャーで：
1. `Ctrl + Shift + Esc` でタスクマネージャーを開く
2. 「詳細」タブを選択
3. PID列で「34020」を探す
4. 右クリック → 「タスクの終了」

#### ステップ2: 開発サーバーを再起動

1. 現在のターミナルで `Ctrl+C` を押してサーバーを停止
2. 以下を実行：

```bash
npm run dev
```

3. 今度はポート3000で起動するはずです
4. `http://localhost:3000` にアクセス

### 方法3: ポートを指定して起動

ポート3000を常に使用したい場合、`package.json`のスクリプトを変更：

```json
"scripts": {
  "dev": "next dev -p 3000"
}
```

ただし、ポート3000が使用中の場合はエラーになります。

## 推奨

**今すぐ確認したい場合** → 方法1（`http://localhost:3001`にアクセス）

**ポート3000を使い続けたい場合** → 方法2（プロセスを停止して再起動）
